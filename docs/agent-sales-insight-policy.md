# AI 销冠回复控制与销售洞察改造说明

## 背景

原会话与客户页面把“风险等级：绿色、黄色、红色”作为主要筛选和展示字段。随着回复策略、人工升级规则、客户标签和会话处理能力逐步完善，风险等级开始承担过多含义：

- 既表示 AI 是否可以自动回复；
- 又表示会话是否需要人工处理；
- 还试图表达客户价值或销售机会。

这会让运营人员难以判断“到底哪个字段决定 AI 是否回复”。因此本次调整把原本混在一起的概念拆开。

## 修改原则

### 1. 回复控制只回答“AI 当前能不能回复”

回复控制由以下信息组成：

- 基础回复策略：自动回复、人工确认、仅记录；
- 好友策略覆盖：好友可以覆盖微信账号默认策略；
- 待确认任务：存在 AI 建议但尚未人工处理；
- 临时接管态：命中人工接管规则后，当前会话暂停自动回复。

这些字段用于控制消息处理流程，不再混入客户价值判断。

命中“人工确认”动作时，系统生成待确认建议和升级原因；命中“人工接管”动作时，系统可以自动把当前会话升级到人工接管，暂停自动回复。

自动升级到人工接管后，系统不会自动改写好友或微信号的回复策略。恢复必须由运营人员在客户工作台点击“恢复原策略”：该动作只清除本次升级产生的临时接管态，让后续消息重新按原本的好友/微信号回复策略执行。

### 2. 销售洞察只回答“这个客户怎么跟进”

销售洞察用于销售管理和客户判断，不直接决定 AI 是否自动发送。首版包含：

- 购买意愿：低、中、高、强烈；
- 销售阶段：新线索、需求确认、产品介绍、报价议价、成交推进、售后咨询；
- 客户情绪：正向、中性、负向；
- 跟进优先级：普通、重点、紧急；
- 客户标签：价格敏感、关注交付、决策人、竞品对比等。

这些信息可以由人工维护，后续也可以由 Agent 根据会话自动识别和推荐。

### 3. 风险等级保留为内部兼容字段

风险等级暂时不从数据库删除，原因是现有人工升级规则、历史会话、回复决策记录和统计逻辑仍然依赖它。首版调整先弱化前端展示，不再把“风险等级”作为销售会话的主筛选。

后续可以逐步把风险等级迁移为“人工升级原因 + 临时回复策略覆盖”。

## 页面改造

### 客户工作台

- 左侧列表从“会话队列”调整为“好友队列”，列表主体是微信好友，打开后进入该好友的会话工作台。
- 队列从“全部 / 待审核 / 风险”调整为“全部 / 待确认 / 人工接管 / 重点跟进 / 紧急跟进”。
- 待确认队列按“存在未处理 AI 建议”判断；人工直接回复后，待确认 AI 建议会标记作废，并从待确认队列移出。
- 重点跟进、紧急跟进按好友的 `follow_up_priority` 判断，人工回复不会自动改变跟进优先级。
- 移除风险等级筛选。
- 右侧面板拆成：
  - 客户信息：客户等级、微信账号、客户编号；
  - 回复控制：会话状态、回复策略、静默秒数、营业时间、连续自动回复；
  - 销售洞察：购买意愿、销售阶段、客户情绪、跟进优先级、客户标签。
- 右侧面板提供编辑入口，支持直接维护客户等级、好友级回复策略、销售洞察和人工标签。
- 会话处于人工接管时，右侧回复控制区显示“恢复原策略”按钮；该按钮不修改回复策略，只恢复到原策略继续执行。

### 客户好友

- 客户好友菜单暂时隐藏，日常运营集中到客户工作台。
- 移除风险等级筛选和风险列。
- 增加销售洞察筛选：购买意愿、销售阶段、跟进优先级。
- 表格增加“销售洞察”列，集中展示洞察字段和客户标签。
- 增加“洞察”编辑入口，支持维护销售洞察字段和客户标签。

## 人工升级规则

人工升级规则统一由 Agent 意图识别结果触发，首版保留三类触发：

- 意图：例如 `refund_or_complaint`、`special_commitment`、`legal_compliance`；
- 情绪：例如 `angry`、`negative`；
- 要求人工：Agent 判断客户明确要求人工介入。

关键词、正则、客户等级、知识库未命中不再作为人工升级规则的直接触发类型。知识库未命中可以先体现在 Agent 意图识别或回复生成的待确认原因中，再由人工决定是否切换策略。

## 数据设计

在 `agent_wechat_contact` 增加以下字段：

- `purchase_intention`：购买意愿，默认 `MEDIUM`；
- `sales_stage`：销售阶段，默认 `NEW_LEAD`；
- `customer_sentiment`：客户情绪，默认 `NEUTRAL`；
- `follow_up_priority`：跟进优先级，默认 `NORMAL`。

客户标签继续使用已有的 `agent_contact_tag` 和 `agent_contact_tag_rel`。

## 默认销售标签

首版预置以下客户标签：

- 价格敏感；
- 关注交付；
- 决策人；
- 竞品对比。

这些标签归属销售洞察区，不参与回复策略自动决策。

## 运营统计

运营统计增加销售洞察维度，首版按好友当前画像做快照统计：

- 购买意愿分布；
- 销售阶段分布；
- 客户情绪分布；
- 跟进优先级分布。

这些统计用于判断当前客户池结构。后续如果要分析趋势，例如“本周高意愿客户增长”，需要增加销售洞察快照或变更日志表。

## 为什么这样改

这样拆分后，运营人员只需要按两个问题理解系统：

1. 当前怎么回复？看回复策略、待确认、人工接管。
2. 这个客户怎么跟进？看销售洞察和客户标签。

风险等级不再承担多个职责，系统语义会更稳定，也方便后续增加 Agent 自动识别购买意愿、销售阶段和客户标签。
