# ai-sales-agent 版本路线图

日期：2026-05-19
所属项目：ai-sales

## 1. 路线图原则

ai-sales-agent 的路线图按“先形成可运营闭环，再增强运营效率，再做增长实验”的顺序推进。每个版本都必须能独立验收，并且每个功能进入开发前都需要从本路线图拆成实施计划。

版本命名采用：

- V1 可运营闭环版。
- V2 复盘优化版。
- V3 协作运营版。
- V4 增长实验版。

## 2. V1 可运营闭环版

V1 是第一版正式后台能力，不再把自动回复、客户标签、敏感规则和知识库放到后续版本。V1 的目标是让一个租户能接入多个 Gewe 托管微信账号，系统接收微信好友消息后，基于 Agent 配置、客户标签、知识库和敏感规则完成回复建议、自动回复或人工接管。

### 2.1 V1 功能范围

- Gewe 微信账号托管接入。
- Gewe 回调接收微信好友消息。
- 微信账号管理，一个租户可管理多个微信账号。
- 微信好友管理，目标客户和重要客户基于微信好友实现。
- 客户标签管理。
- 会话与消息记录。
- LLM 回复建议。
- 自动回复。
- 人工审核和人工接管。
- 敏感规则配置。
- 红色风险暂停自动回复。
- 知识库管理。
- Agent 基础配置、回复策略、版本发布记录。

### 2.2 V1.1 接入闭环

目标：微信账号、好友、会话和消息可以稳定入库。

功能：

- 新建 `sales-module-agent`。
- 新增 `AI 销冠` 后台菜单。
- 新增微信账号管理页面和接口。
- 为每个微信账号生成 Gewe 回调地址和回调令牌。
- 实现 Gewe 回调接口。
- 保存 Gewe 原始回调事件。
- 实现回调事件幂等处理。
- 保存或更新微信好友。
- 创建或更新会话。
- 保存客户消息。
- 会话记录后台可查询。

验收标准：

- 可以在后台创建微信账号接入配置。
- 可以复制回调地址配置到 Gewe。
- 模拟 Gewe 文本消息回调后，系统保存原始事件、好友、会话和消息。
- 同一回调事件重复推送不会生成重复消息。
- 不同租户的数据互不可见。

### 2.3 V1.2 Agent 决策闭环

目标：客户消息进入系统后，Agent 可以生成回复建议，并由人工审核发送。

运营主题：运营可控。V1.2 先补齐“可配置、可审核、可干预”的运营闭环，不把风险看板和发布复盘提前合并进来。

功能：

- Agent 基础配置。
- Agent 回复模式：自动回复、人工确认、仅记录。
- LLM 回复建议服务抽象。
- 构造会话上下文。
- 保存回复决策记录。
- 回复审核列表。
- 审核通过并发送。
- 修改建议后发送。
- 拒绝建议并记录原因。
- 保存 AI 或人工回复消息。

验收标准：

- 客户消息进入后可以生成回复建议。
- 后台可以查看待审核回复。
- 审核通过后调用 Gewe 发送文本消息。
- 修改后发送保留修改人和修改后内容。
- 拒绝后不会发送消息，但保留决策和审计记录。

### 2.4 V1.3 自动回复与风控

目标：系统可以安全地自动回复普通消息，并在风险场景暂停自动回复。

运营主题：风险可视。V1.3 把 V1.2 产生的规则命中、风险等级、接管状态可视化，形成风险运营台。

功能：

- Agent 自动回复开关。
- 置信度阈值。
- 单客户连续自动回复上限。
- 营业时间和静默时间控制。
- 敏感规则管理。
- 规则类型：关键词、正则、LLM 分类。
- 风险等级：绿色、黄色、红色。
- 风险动作：允许、人工确认、暂停并接管。
- 红色风险会话创建接管任务。
- 人工接管状态下暂停自动回复。
- 风险会话列表和上下文抽屉。

验收标准：

- 满足自动回复条件的普通消息可以自动发送。
- 置信度不足时进入人工确认。
- 超过连续自动回复上限时进入人工确认。
- 命中红色规则时不会自动发送。
- 红色风险会话可以标记人工接管。

### 2.5 V1.4 运营配置

目标：运营人员可以维护客户分层、知识库和 Agent 配置版本。

运营主题：复盘发布。V1.4 承接 V1.2 的回复决策记录和 V1.3 的风险记录，重点解决“为什么这么回、效果如何、配置如何发布和追溯”。

功能：

- 客户标签管理。
- 好友绑定多个客户标签。
- 目标客户和重要客户标记。
- 知识源管理。
- 知识源类型：文本、URL、FAQ。
- 知识源启停。
- Agent 绑定知识源。
- 回复决策记录知识引用。
- 保存 Agent 草稿。
- 发布 Agent 配置。
- 配置版本历史。
- 基础统计：今日消息数、自动回复数、待审核数、风险会话数。

验收标准：

- 运营人员可以为微信好友打标签。
- 客户标签进入 LLM 上下文。
- 启用知识源后，回复决策记录包含知识引用。
- Agent 配置可以保存草稿和发布。
- 发布后可以查看版本历史。

## 3. V2 复盘优化版

V2 目标是让系统从“能运营”进化到“能自我改进”。重点是复盘、建议和策略调整任务。

功能：

- 每日复盘生成。
- 今日摘要：消息量、自动回复量、人工接管量、风险等级分布。
- 表现较好的回复策略分析。
- 高频客户问题统计。
- 知识缺口识别。
- 敏感规则命中分析。
- 调整任务生成。
- 调整任务关联目标模块：知识库、敏感规则、回复策略、客户标签。
- 调整任务确认或撤回。
- 确认后的调整进入 Agent 草稿。
- Skill 挂载管理。

验收标准：

- 每天可以生成一份 Agent 复盘。
- 复盘至少包含一条可操作调整建议。
- 确认调整后，系统能把变更写入草稿而不是直接发布。
- 发布历史能追溯复盘建议带来的配置变化。

## 4. V3 协作运营版

V3 目标是支持销售团队协作和处理 SLA，让 AI 与人工销售形成稳定协同。

功能：

- 销售团队和负责人分配。
- 客户归属规则。
- 风险会话处理 SLA。
- 超时提醒。
- 飞书、企业微信或站内信提醒。
- 人工接管任务流转。
- 临时接管态和“恢复原策略”操作记录。
- 会话备注和内部协作记录。
- 会话质检评分。
- 人工回复质量审计。

验收标准：

- 红色风险会话能分配给负责人。
- 超过 SLA 未处理会生成提醒。
- 销售主管可以查看团队人工接管会话和恢复原策略记录。
- 每条人工接管记录可追溯触发原因、处理人、恢复时间和恢复前后的原始策略来源。

## 5. V4 增长实验版

V4 目标是支持多策略实验和效果评估，让 Agent 具备持续增长能力。

功能：

- 多 Agent 策略实验。
- 回复策略 A/B 测试。
- 客户阶段自动识别。
- 客户意向评分。
- 回复效果评分。
- 成交线索标记。
- 模型供应商路由。
- 模型成本统计。
- 自动回复效果看板。
- 不同知识库版本效果对比。

验收标准：

- 同一类客户可以分配到不同策略组。
- 后台可以比较不同策略组的回复质量和人工接管率。
- 模型调用量、失败率和成本可统计。
- 运营人员可以基于实验结果决定是否发布新策略。

## 6. Backlog 管理规则

所有后续需求进入开发前，需要先归入以下状态之一：

- `Candidate`：候选需求，已记录但不承诺版本。
- `Planned`：已进入某个版本计划。
- `Designing`：正在写设计文档。
- `Ready`：设计已确认，可以写实施计划。
- `Building`：正在开发。
- `Verified`：开发完成并通过验收。

每个需求至少记录：

- 需求名称。
- 所属版本。
- 用户价值。
- 业务流程影响。
- 涉及数据对象。
- 验收标准。

## 7. 当前优先级

当前只推进 V1。V2、V3、V4 作为已记录路线图，不进入本轮开发。

V1 开发顺序固定为：

1. V1.1 接入闭环。
2. V1.2 Agent 决策闭环。
3. V1.3 自动回复与风控。
4. V1.4 运营配置。

每个阶段完成后需要先演示并确认，再进入下一阶段。
